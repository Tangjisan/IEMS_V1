# IEMS_V1 项目说明

## 1. 项目概述

IEMS_V1（Intelligent Environmental Monitoring System Version 1）是基于 STM32F1xx 系列微控制器开发的智能环境监测系统，专注于实现多维度环境参数的实时采集、数据处理与可视化展示。系统集成了多种主流传感器与硬件模块，支持通过中断、DMA 等高效数据传输方式，确保环境数据采集的准确性与实时性，可广泛应用于室内外环境监测、智能家居、工业环境监控等场景。

## 2. 硬件环境

### 2.1 核心控制器

采用 STM32F1xx 系列微控制器，依托 STM32CubeHAL 库提供的底层驱动支持，实现对各类外设的统一管理与调度，核心时钟通过 HSE 外部晶振（8MHz）结合 PLL 倍频（9 倍频）配置为 72MHz，为系统提供稳定高效的运行基础。

### 2.2 传感器模块

系统集成多种环境传感器，覆盖空气质量、温湿度、光照、气压等核心监测维度，具体参数如下：

| 传感器型号 | 监测参数                                         | 通信接口 | 核心功能                                       |
| :--------- | :----------------------------------------------- | :------- | :--------------------------------------------- |
| SGP30      | eCO2（等效二氧化碳）、TVOC（总挥发性有机化合物） | I2C      | 气体质量监测，支持 CRC8 数据校验               |
| AHT20      | 温度、湿度                                       | I2C      | 高精度温湿度采集，支持 DMA 数据传输            |
| BMP280     | 气压、温度                                       | I2C      | 高分辨率气压检测，支持多种过采样模式与滤波配置 |
| BH1750     | 光照强度（lux）                                  | I2C      | 宽范围光照检测，支持自动 / 单次测量模式        |
| PMS5003    | PM1.0、PM2.5、PM10                               | UART     | 颗粒物浓度监测，支持自动 / 强制数据发送模式    |

### 2.3 辅助硬件模块

- **编码器（Encoder）**：通过 GPIO 外部中断实现旋转计数，支持正反转识别与防抖处理（防抖时间 5ms），可用于设备参数调节。
- **OLED 显示屏**：采用 I2C 通信接口，分辨率 128×64，支持字符、字符串、数字（十进制 / 十六进制 / 二进制）的显示，用于实时展示传感器数据。
- **SDHC 存储（预留）**：通过 SPI 接口实现 SD 卡数据存储，预留数据本地缓存功能扩展。
- **通信接口**：包含 UART（USART2，波特率 9600）、I2C（I2C2）、SPI（SPI1），支持传感器数据传输与外部设备通信。

## 3. 软件架构

### 3.1 系统分层设计

系统采用模块化、分层设计思想，自上而下分为应用层、驱动层、硬件抽象层，各层职责清晰、低耦合：

- **应用层**：包含任务管理模块（如`Task_test_1.cpp`），负责传感器数据的整合、处理与展示逻辑，通过调用驱动层接口实现业务功能。
- **驱动层**：为各类硬件提供统一的操作接口，封装传感器初始化、数据读取、参数配置等核心功能，每个硬件模块独立命名空间（如`n_SGP30`、`n_OLED`），避免命名冲突。
- **硬件抽象层**：基于 STM32CubeHAL 库，封装 GPIO、I2C、UART、DMA、TIM 等外设的底层操作，为驱动层提供统一的硬件访问接口。

### 3.2 核心软件模块

1. **传感器驱动模块**
   - 统一提供`init()`（初始化）、`readData()`（数据读取）、`n_get`（数据获取接口）等标准化接口，降低应用层调用复杂度。
   - 支持两种数据传输模式：轮询模式（适用于低频率采集）和 DMA 模式（适用于高频率、大数据量采集，减少 CPU 占用）。
   - 内置数据校验机制（如 SGP30 的 CRC8 校验、PMS5003 的帧校验码验证），确保数据可靠性。
2. **OLED 显示模块**
   - 支持多格式数据显示，包括 ASCII 字符、字符串、无符号数、有符号数、十六进制数、二进制数。
   - 提供光标位置设置、清屏等基础操作，支持 4 行 16 列的固定布局显示，满足多参数同时展示需求。
3. **中断与 DMA 管理**
   - 中断处理：编码器采用 EXTI 外部中断（EXTI15_10_IRQn），定时器采用更新中断（TIM2、TIM3），UART/I2C 采用传输完成中断，确保事件响应的实时性。
   - DMA 传输：I2C2、USART2 均配置 DMA 通道，实现传感器数据的高速传输，降低 CPU 负载，支持外设到内存、内存到外设的双向数据搬运。
4. **任务管理模块**
   - 通过`cpp_main()`函数作为 C++ 任务入口，支持多任务扩展（预留`n_task_test_1`至`n_task_test_5`任务命名空间）。
   - 核心任务逻辑：初始化硬件→配置传感器→循环采集数据→数据校验→OLED 显示，支持 1 秒周期的数据更新。

## 4. 核心功能

### 4.1 环境数据采集

- 多维度参数覆盖：支持 eCO2、TVOC、温湿度、光照强度、气压、颗粒物浓度（PM1.0/PM2.5/PM10）的同步采集。
- 灵活采集模式：传感器支持自动采集与手动触发采集，可通过配置切换（如 PMS5003 的自动 / 强制发送模式、BH1750 的自动 / 单次测量模式）。

### 4.2 数据处理与校验

- 数据转换：将传感器原始数据转换为物理量（如 AHT20 湿度转换为 0-100% RH，温度转换为 - 50~150℃）。
- 校验机制：通过 CRC 校验、帧格式校验、数据范围校验等多重方式，过滤无效数据，确保采集精度。

### 4.3 数据可视化

通过 OLED 显示屏实时展示各传感器数据，界面布局清晰，支持参数标签与数值的对应显示，便于直观查看环境状态。

### 4.4 硬件控制与扩展

- 编码器控制：支持通过编码器旋转计数实现参数调节（如传感器采集频率、OLED 显示亮度等，预留扩展接口）。
- 存储扩展：预留 SDHC 存储接口，可扩展数据本地存储功能，支持历史数据回溯。

## 5. 使用说明

### 5.1 系统初始化

- 硬件上电后，系统自动执行`main()`函数，依次初始化 GPIO、DMA、I2C、UART、TIM 等外设。
- 调用`cpp_main()`启动应用层任务，初始化传感器与 OLED 显示屏，完成系统就绪配置。

### 5.2 数据查看

系统初始化完成后，OLED 显示屏将按固定布局循环刷新各传感器数据，默认显示周期为 1 秒，数据格式如下：

- 第0页：封面
- 第1页：传感器AHT20：温度（单位：℃）、湿度（单位：% RH）
- 第2页：传感器BH1750：光照强度（lux）
- 第3页：传感器BMP280：温度（单位：℃）或气压（单位：Pa）
- 第4页：传感器SGP30：eCO2 浓度（单位：ppm）、TVOC 浓度（单位：ppb）
- 第5页：传感器PMS5003： PM1.0 浓度（单位：μg/m³）、PM2.5（单位：μg/m³）、PM10（单位：μg/m³）

### 5.3 参数配置

可通过编码器或预留的 UART 接口修改系统参数，如传感器采集频率、OLED 显示亮度、数据发送模式等，具体配置需结合应用层任务逻辑扩展。

### 6.3 关键共享资源说明

#### I2C2 总线共享

- 涉及传感器：AHT20（温湿度）、SGP30（空气质量）、BH1750（光照）、BMP280（气压）
- 共享 IO 口：PB10（SCL）、PB11（SDA）
- 冲突避免：通过 “分时复用” 机制，同一时间仅与一个传感器通信；驱动中加入 I2C 总线就绪等待（_wait_i2c_ready），确保总线空闲后再发起通信。

#### DMA1 通道共享

- 通道 4：I2C2_TX、USART2_TX
- 通道 5：I2C2_RX、USART2_RX
- 通道 6：USART2_RX（PMS5003）
- 通道 7：USART2_TX（PMS5003）
- 冲突避免：通过状态机（如 CbState_t）管理通道占用状态，仅当通道空闲时发起新的 DMA 传输；优先级配置（DMA_PRIORITY_LOW）确保核心传感器数据传输优先。

#### 定时器共享

- TIM3：AHT20、SGP30、BH1750、BMP280 的 DMA 模式采集触发
- TIM2：预留扩展（如 PMS5003 强制模式定时采集）
- 配置说明：TIM3 初始化周期为 5ms（Prescaler=7199，Period=4999），通过定时中断触发传感器数据接收，确保各传感器采集节奏一致。

#### 补充说明

1. 所有 IO 口均在`gpio.c`中统一初始化，传感器驱动仅负责通信逻辑，无需重复配置引脚，保证配置一致性。
2. 传感器与外设的关联均通过 “命名空间封装” 实现（如`n_AHT20`关联 I2C2），降低模块间耦合，便于维护与扩展。
3. 中断优先级配置：EXTI（编码器）> 定时器（采集触发）> DMA（数据传输）> I2C/USART（通信），确保关键操作优先响应。

## 6. 扩展方向

- **数据上传**：通过 UART 或新增 Wi-Fi 模块（如 ESP8266），实现传感器数据上传至云平台或手机 APP，支持远程监控。
- **报警功能**：添加 GPIO 控制的蜂鸣器 / LED 报警模块，当环境参数超出预设阈值时触发报警。
- **数据存储**：完善 SDHC 存储驱动，实现传感器数据的定时存储与历史数据查询。
- **多传感器扩展**：基于现有 I2C/SPI 接口，扩展更多类型传感器（如甲醛传感器、CO 传感器），丰富监测维度。

## 7. 注意事项

- 传感器初始化需严格遵循通信时序要求，部分传感器（如 AHT20）上电后需等待 40ms 稳定时间再执行初始化操作。
- DMA 模式使用前需确保对应的 DMA 通道已正确配置（如 I2C2 的 DMA 接收通道为 DMA1_Channel5，发送通道为 DMA1_Channel4）。
- PMS5003 传感器需注意 UART 波特率匹配（默认 9600），数据帧长度为 32 字节，需完整接收后再进行校验解析。
- 系统运行过程中若出现数据异常，可通过 OLED 显示的 "err" 标识定位故障模块，优先检查传感器接线与通信接口配置。
